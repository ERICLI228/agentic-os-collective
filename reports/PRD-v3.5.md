## 📜 v3.4 → v3.5 核心修订摘要

| 维度 | v3.4（旧） | v3.5（新） |
|------|-----------|-----------|
| **核心问题** | 用更好的执行层解决问题 | **先建感知控制层，再优化执行层** |
| **产品愿景** | "对抗审核闭环" | **"关键节点等你决策"** — 你不在，系统不走 |
| **建设顺序** | Stage 1-4 品类→数据→流程→团队 | **Sprint 0-4: 感知→决策→执行→优化** |
| **决策节点** | 部分 milestone 有 decision_point | **三个明确硬节点：市场判断/选品审核/发布审批** |
| **发布控制** | 靠"不跑脚本"防止误发布 | **publish_gate 硬约束，代码级别阻止未批准发布** |
| **驾驶舱** | 壳子在，数据空 | **Sprint 0 先建状态API → Sprint 1 决策界面 → Sprint 2 完整看板** |
| **飞书交互** | 单向推送（只看不能点） | **双向决策卡片（看到数据 + 点通过/修改/驳回）** |
| **红线规则** | 未明确 | **5 条硬红线：不自动发布/不跳过决策/不硬编码密钥/不跨线调度/不覆盖数据** |
| **执行计划** | Stage 1-4 四阶段串行 | **Sprint 0-4 五阶段，Sprint 0 止血(感知优先)** |

## 第一部分：产品概述

### 1.1 产品定位

Agentic OS v3.5 是一个**基于智能体架构的双业务线自动化中台**，通过统一基座管理 TK 跨境电商运营和 AI 数字短剧制作两条独立业务线。产品以"**一个指令启动、全程自动执行、关键节点等你决策**"为核心体验。

**核心认知**: 系统再怎么自动，只要你还承担最终责任，关键决策就必须由你做出。Agent 可以做分析、做推荐、做准备，但"值不值得做""选哪个上架""发不发"这三个问题，必须等你点头。

## 第三部分：三个决策节点（核心设计）

这是 v3.5 最核心的设计。**系统必须在这三个节点停下来等你，不等到你的信号绝不往下执行。**

### 节点A：市场判断

```
触发时机: 数据采集完成，发现潜在品类机会
你看到的: 五维数据卡片 — 竞品饱和度、利润空间、供应链、季节性、达人潜力
你决策的: 🔍 值得做 / 📝 换品类 / ❌ 不做
未决策后果: 系统暂停，每天飞书提醒一次
关联模块: FR-TK-001 → FR-TK-016
```

### 节点B：选品审核

```
触发时机: Agent 完成选品分析，推荐 TOP3-5 SKU
你看到的: 推荐排行 — 每商品1688价格/预估利润/竞品数量/对抗审核评分
你决策的: ✅ 通过(推TOP3) / ✏️ 修改条件(重跑) / ❌ 驳回(说明原因)
未决策后果: 系统暂停，飞书每日提醒
关联模块: FR-TK-003→006 → FR-TK-016
```

### 节点C：发布审批

```
触发时机: 商品已准备好上架到 TK 店铺
你看到的: 发布预览 — 商品名/店铺/售价/预估利润/预览图
你决策的: ✅ 批准发布 / 🧪 仅测试1条 / ❌ 拒绝
未决策后果: 绝不发布，哪怕等一个月
关联模块: FR-TK-007
```
