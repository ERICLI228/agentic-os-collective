## 三大决策节点（硬约束，不绕过）

```
节点A: 市场判断 ──── 采集前，你看数据点"值得做"→ 才触发采集
节点B: 选品审核 ──── Agent推荐3-5SKU，你选哪个/改什么价 → 才上架
节点C: 发布审批 ──── 内容预览，你点"批准"→ 才发布到TK（支持"测试发1条"）
```

**规则**: 任何 Agent 到达这3个节点，必须停下来等你决策。不等到你的信号，绝不往下执行。这条规则写进 `shared/knowledge/best_practices.yaml` 和 CLAUDE.md。

### Task 0.2: 建选品分析引擎 + 飞书决策卡片

**目标**: 从采集箱拉50商品 → 分析利润/竞争度 → 推飞书卡片 → 你在飞书决策

**子任务**:
- [ ] 0.2.1 创建 `tk/openclaw/core/tk_selection_analyzer.py`
  - 从妙手采集箱拉取50商品（已有API可用）
  - 计算每商品: 1688价格、预估运费、TK佣金、利润空间
  - 按利润排序TOP5
  - 输出结构化JSON（供卡片渲染）
- [ ] 0.2.2 创建 `shared/feishu_cards/selection_card.py`
  - 飞书交互式卡片模板（五维数据 + 利润排行 + 建议）
  - 三个按钮: [通过(推TOP5)] [修改(调筛选条件)] [驳回(说明原因)]
- [ ] 0.2.3 实现飞书卡片回调Webhook
  - 接收飞书按钮点击事件
  - "通过" → 进入选品审核流程（推下一张卡片/记录决策）
  - "修改" → 重新跑分析（带修改参数）
  - "驳回" → 记录原因，标记任务终止
- [ ] 0.2.4 集成到触发流程（目前手动触发，后续改定时）

**验收标准**: 你发一句"做选品分析"，飞书收到带数据的分析卡片，你能点按钮决策
