### 3.2 TK 运营功能详细说明

#### FR-TK-001: 核心品类数据采集

| 属性 | 内容 |
|------|------|
| **优先级** | P0 |
| **触发方式** | Cron 定时 (每 2 小时) |
| **执行角色** | Worker1 选品专家 |
| **输入** | 国家：印尼/越南/泰国/菲律宾/马来西亚，**品类：先验证 2 个 3C 子品类** |
| **处理逻辑** | 调用 TikTok API/FastMoss 获取热门视频列表，统计播放量/点赞/评论/分享，提取商品链接和价格信息 |
| **输出** | 品类分布饼图、原始数据 Excel (存储在 `artifacts/`) |
| **异常处理** | API 返回空时重试 3 次，仍失败则标记 MS-1 为 failed 并飞书告警 |
| **验收标准** | 单次采集<5 分钟，采集成功率>95% |

> **🆕 v3.4 调整**: 从 16 品类改为先验证 2 个核心品类。验证通过后再逐步扩展。

| 属性 | 内容 |
|------|------|
| **优先级** | P1 |

| **决策点** | ⚠️ ROI<100% 时创建决策点：暂停广告/调整预算/忽略 |

| 后端框架 | Python Flask | 已有代码基础，团队熟悉 |
| 前端框架 | Vue 3 + Vite | 已部署，组件化开发 |
| 进程守护 | pm2 | macOS 兼容，简单可靠 |
| 模型路由 | 本地优先→云端降级 | 成本最低原则 |
| 跨 APP 调度 | MCP 协议 | 标准化接口，解耦执行端 |
