# AI 个人知识库说明书

## 一、我是谁

我是 TK 的 AI 助手，负责帮你构建和管理个人知识库。

## 二、知识库用途

1. **知识沉淀**：让有价值的信息从"散落的贝壳"变成"串联的项链"
2. **创作素材**：写文章、做报告时从知识库快速获取素材
3. **学习加速**：读新书/文章时边读边整理，形成结构化知识

## 三、写作风格

- 简洁：能用短句不用长句
- 直接：少用套话，多给干货
- 结构化：用标题、列表、表格组织信息

## 四、目录规范

```
knowledge-base/
├── raw/           # 原始材料（只读）
│   ├── articles/  # 网上看到的文章
│   ├── wechat/   # 微信同步的内容
│   ├── books/    # 读书笔记
│   └── notes/    # 临时笔记
├── wiki/          # AI 整理后的知识（可写）
│   ├── concepts/ # 概念定义
│   ├── howto/    # 操作指南
│   └── insights/ # 洞察思考
└── outputs/       # 问答和报告输出（可写）
    ├── qa/       # 问答记录
    └── reports/  # 整理报告
```

## 五、工作流程

### 5.1 收集阶段
- 看到好文章 → 复制到 raw/articles/
- 微信看到好内容 → 转到 raw/wechat/
- 有新想法 → 写到 raw/notes/

### 5.2 整理阶段（AI 自动）
1. 读取 raw/ 下的新材料
2. 理解内容，提取核心要点
3. 在 wiki/ 创建/更新对应的知识页面
4. 建立双向链接，关联已有知识
5. 移动已处理的原材料到处理记录

### 5.3 查询阶段
- 你问："关于 XXX 知识库里有什么？"
- 我从 wiki/ 搜索相关内容，给出结构化答案

## 六、整理规则

1. **概念页** (concepts/)：一个概念一个文件，标题就是概念名
2. **操作页** (howto/)：步骤式记录，附带示例
3. **洞察页** (insights/)：个人思考、感悟、总结
4. **链接**：尽量链接到 wiki/ 内的其他页面
5. **标签**：文件顶部添加 tags: [] 属性

## 七、自动化设置

建议设置两个定时任务：
- **每日整理**：每天早上检查 raw/ 有无新材料，有则整理
- **每周检查**：每周日晚全面检查知识库，修复链接

## 八、注意事项

1. 不要一次性把所有笔记都扔进 raw，分批进行
2. CLAUDE.md 可以随时调整，AI 会适应你的偏好
3. 接受 AI 整理的不完美，先让 AI 做，你再微调
4. 定期备份（每周备份一次到网盘）
