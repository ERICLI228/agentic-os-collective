## MANDATORY: Agent Workflow

每个新会话必须遵循此工作流：

### Step 1: 读取任务

读取 `task.json`，选择一个任务执行。

选择标准（按优先级）：
1. 选择 `passes: false` 的任务
2. 考虑依赖关系 - 基础功能优先
3. 选择最高优先级的未完成任务

### Step 2: 实现任务

- 仔细阅读任务描述和步骤
- 按照步骤实现功能
- 遵循现有代码模式

### Step 3: 测试验证

**强制测试要求：**

1. **大幅度修改**（新增功能、修改核心逻辑）：
   - 必须验证功能可用
   - 检查日志输出

2. **小幅度修改**（修复bug、调整参数）：
   - 验证无报错

3. **所有修改必须通过**：
   - 脚本语法检查通过
   - API调用成功（如适用）
   - 文件正确生成

### Step 4: 更新进度

将工作记录到 `progress.txt`：

```
## [日期] - 任务: [任务名称]

### 完成内容:
- [具体修改]

### 测试:
- [如何测试]

### 备注:
- [对后续agent的说明]
```

### Step 5: 提交更改

更新 `task.json`，将任务的 `passes` 从 `false` 改为 `true`（仅当所有步骤验证通过后）。

## ⚠️ 阻塞处理（Blocking Issues）

**如果任务无法完成，必须遵循以下规则：**

### 需要停止任务并请求人工帮助的情况：

1. **缺少配置**：
   - API Key 需要填入
   - TOS 配置需要创建
   - 外部服务需要开通账号

2. **外部依赖不可用**：
   - 第三方 API 服务宕机
   - 网络连接问题
   - 配额不足

3. **需要人工授权**：
   - 飞书审批
   - 人工审核剧本

### 阻塞时的正确操作：

**DO NOT（禁止）：**
- ❌ 跳过任务继续下一步
- ❌ 假装任务已完成
- ❌ 删除或修改任务描述

**DO（必须）：**
- ✅ 在 progress.txt 中记录当前进度和阻塞原因
- ✅ 输出清晰的阻塞信息，说明需要人工做什么
- ✅ 停止任务，等待人工介入

### 阻塞信息格式：

```
🚫 任务阻塞 - 需要人工介入

**当前任务**: [任务名称]

**已完成的工作**:
- [已完成的代码/配置]

**阻塞原因**:
- [具体说明为什么无法继续]

**需要人工帮助**:
1. [具体的步骤 1]
2. [具体的步骤 2]
...

**解除阻塞后**:
- 运行 [命令] 继续任务
```

## 命令

```bash

python3 water_margin_drama.py "武松打虎"

# 仅生成配音
python3 drama_audio.py --tts "台词内容"

# 合并音视频
python3 drama_audio.py --merge video.mp4 audio.m4a

# 完整流水线
python3 drama_audio.py --full "视频URL" "配音文字" "输出名称"
```

## 关键规则

1. **一次一任务** - 专注于完成一个任务
2. **测试后再标记完成** - 所有步骤必须验证通过
3. **阻塞时停止** - 需要人工介入时，输出阻塞信息并停止
4. **记录到progress.txt** - 帮助后续agent理解你的工作
5. **不删除任务** - 只能将 `passes` 从 `false` 改为 `true`
6. **按顺序执行** - 注意任务依赖关系
